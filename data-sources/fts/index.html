Monitoring @UC

Index pattern is fts-2*.

To check queue lenghts and average time in queue, check this dashboard: http://atlas-kibana-dev.mwt2.org/goto/5132d543e93e074899b0b92aa3cf1265

To check number of active transfers per activity type and per link, use index pattern links_traffic*.

To check ingress, egress, numbers of incomming and outcoming transfers per site, use index pattern sites_traffic* and this dashboard: http://atlas-kibana-dev.mwt2.org/goto/608b3f7a29b7b37afdd50a2c62fdd12c

Monitoring flow description

The new data flow started March 2018. Data is collected from the FTS AMQ and sent directly to ES at UC, so it is almost real-time. All the IDs are stored so results can be crosschecked against both RUCIO and FTS.
